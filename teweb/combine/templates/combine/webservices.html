REST web services

Web Services provide access to tellurium-web in a language independent fashion. To use them programmatically, developers can create their own client application to automate the access to the database directly from their simulation software, tools or databases. tellurium-web can be automatically accessed via RESTful Web Services via HTTP requests. COMBINE archives can be accessed using simple http GET requests to either retrieve a complete COMBINE archive. The entry point of the API is available from here. The REST API provides access the COMBINE archives, tags and users of Tellurium Web.

REST endpoints (urls)

  • /api/: API entry point and schema (swagger)
  • /api/archives/: access COMBINE archives
  • /api/archive/{uuid}: access single COMBINE archive
  • /api/tags/: access tags for COMBINE archives
  • /api/tag{uuid}: access single tag
  • /api/users/: access users
  • /api/user/{uuid}: access single user

Authentication

For a subset of the REST functionality a login is required. Login with your tellurium-web credentials at `./api/` To authenticate via python use the following code snippet 
# base url
base_url = http://127.0.0.1:8001

# authentication
auth = coreapi.auth.BasicAuthentication(
    username='mkoenig',
    password='*******'
)
client = coreapi.Client(auth=auth)
document = client.get(base_url + "/api/")
